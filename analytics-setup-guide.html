📊 SmartBrief AI — Analytics Setup Guide

Connect your static site to a Google Sheet in under 10 minutes. No coding after setup.

What this gives you:

Every page view, button click, specialty switch, and ROI slider interaction gets logged as a row in your Google Sheet — automatically, silently, and for free. No monthly fee. No cookie banner. No external service.

1 Open Google Apps Script

Go to script.google.com in any browser (must be logged into your Google account).

[ Screenshot: You'll see the Google Apps Script home page with "New project" button in the top-left corner ]

Click "New project". A code editor opens with a default function myFunction() {} stub.

2 Create a linked Google Sheet

In the Apps Script editor, click the spreadsheet icon in the left sidebar (or go to Services → Sheets). Alternatively:

  • Open Google Sheets at sheets.google.com
  • Create a new blank spreadsheet named "SmartBrief AI Analytics"
  • Leave the first sheet as "Sheet1" (do not rename it)
  • In row 1, add these column headers exactly:
    timestamp
    event_type
    page
    specialty_selected
    section_viewed
    cta_clicked
    referrer
    user_agent
    session_id
  • Back in Apps Script: click Extensions → Apps Script from the Sheets menu bar. This opens the script editor already bound to your sheet.
[ Screenshot: Google Sheets menu bar → Extensions → Apps Script → opens bound script editor ]

3 Paste the Apps Script code

In the Apps Script editor, select all existing code (Ctrl+A / Cmd+A) and replace it entirely with the code below:

function doPost(e) {
  try {
    var sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
    var data = JSON.parse(e.postData.contents);
    sheet.appendRow([
      new Date().toISOString(),
      data.event_type || '',
      data.page || '',
      data.specialty_selected || '',
      data.section_viewed || '',
      data.cta_clicked || '',
      data.referrer || '',
      data.user_agent || '',
      data.session_id || ''
    ]);
    return ContentService
      .createTextOutput(JSON.stringify({status: 'ok'}))
      .setMimeType(ContentService.MimeType.JSON);
  } catch(err) {
    return ContentService
      .createTextOutput(JSON.stringify({status: 'error', message: err.toString()}))
      .setMimeType(ContentService.MimeType.JSON);
  }
}

Click the 💾 Save icon (or Ctrl+S). Name the project "SmartBrief Analytics" when prompted.

4 Deploy as a Web App

In the Apps Script editor toolbar, click "Deploy" → "New deployment".

[ Screenshot: "Deploy" dropdown menu in top-right of editor, "New deployment" option highlighted ]
  • Click the gear icon ⚙️ next to "Select type" → choose "Web app"
  • Description: SmartBrief Analytics v1
  • Execute as: Me (your Google account)
  • Who has access: Anyone
[ Screenshot: New deployment dialog — "Web app" type selected, Execute as "Me", Who has access "Anyone" ]

Click "Deploy". Google will ask you to authorise the app — click "Authorise access", choose your Google account, and click "Allow".

⚠️ You may see a "Google hasn't verified this app" warning. Click "Advanced" → "Go to SmartBrief Analytics (unsafe)" → "Allow". This is normal for your own scripts.

After deploying, you will see a Web app URL that looks like:
https://script.google.com/macros/s/AKfycb.../exec

Copy this URL — you will need it in the next step.

[ Screenshot: Deployment successful dialog with the Web app URL displayed and a "Copy" button ]

5 Paste the URL into your HTML files

Open each of the three HTML files in a text editor (or in your GitHub repository):

  • index.html
  • smartbrief-dashboard.html
  • smartbrief-patient.html

In each file, near the bottom (just before </body>), find this exact line:

  // ========== PASTE YOUR GOOGLE APPS SCRIPT URL BELOW ==========
  var ANALYTICS_ENDPOINT = 'YOUR_GOOGLE_APPS_SCRIPT_URL_HERE';
  // ===============================================================

Replace YOUR_GOOGLE_APPS_SCRIPT_URL_HERE with your deployment URL:

  // ========== PASTE YOUR GOOGLE APPS SCRIPT URL BELOW ==========
  var ANALYTICS_ENDPOINT = 'https://script.google.com/macros/s/AKfycb.../exec';
  // ===============================================================

Save the file, commit, and push to GitHub. Analytics are now live.

[ Screenshot: VS Code or GitHub web editor showing the ANALYTICS_ENDPOINT line with the URL pasted in ]

📋 What gets tracked

Once live, your Google Sheet will automatically receive a new row for each of these events:

  • page_view — every time any of the three pages loads
  • cta_click — when a visitor clicks a WhatsApp button, Open Dashboard, or pricing/beta links
  • specialty_switch — when a visitor clicks Dental / ENT / Gynae / General on the landing page
  • pricing_toggle — when a visitor switches between Monthly and Annual pricing
  • roi_slider_change — when a visitor adjusts the ROI calculator (debounced, 1 event per slider stop)
  • section_view — when a visitor scrolls to a major section (Who, Compare, ROI, Pricing, Beta)

🛡️ Privacy & safety

  • No cookies, no fingerprinting, no PII (names, phone numbers, patient data)
  • Only anonymous session IDs stored in sessionStorage (cleared on tab close)
  • All tracking silently disabled when the site is opened locally (file: or content: protocol)
  • If the Apps Script URL is unreachable, the site works exactly as normal — no errors, no broken buttons

🔄 Updating the script later

If you make changes to the Apps Script code, you must create a new deployment — editing an existing deployment does not update the live URL. Go to Deploy → New deployment, copy the new URL, and update the ANALYTICS_ENDPOINT in your HTML files again.
